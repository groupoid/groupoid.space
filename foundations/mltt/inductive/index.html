INDUCTIVE BASIS

Published: 27 JAN 2022

These types form the inductive basis of MLTT foundations and are built-in into type checker.

EMPTY

The Empty type represents False-type logical 𝟎, type without inhabitants, Void or ⊥ (Bottom). As it has not inhabitants it lacks both constructors and eliminators, however, it has induction.

Definition (Empty-type 𝟎). Empty-type is defined as built-in 𝟎-type. $$ 𝟎 : U $$

def Empty : U := 𝟎

Induction Principle

Definition (Induction Principle $ind_0$). 𝟎-type is satisfying the induction principle: $$ \prod_{C\ :\ 𝟎\ \rightarrow\ U} \prod_{z\ :\ 𝟎} C(z) := ind_0(C(z),z). $$


def Empty-ind (C: 𝟎 → U) (z: 𝟎) : C z := ind₀ (C z) z

Definition (Negation or isEmpty). For any type A negation of A is defined as arrow from A to 𝟎. $$ \neg A := A \rightarrow 𝟎. $$

def isEmpty (A: U): U := A → 𝟎

The witness of $\neg A$ is obtained by assuming A and deriving a contradiction. This techniques is called proof of negation and is constructive in constrast to non-constructive proof by contradiction which implies $\neg\neg A \rightarrow A$ (double negation elimination).


UNIT

Unit type is the simplest type equipped with full set of MLTT inference rules. It contains single inhabitant ★ (Star).

Definition (Unit-type 𝟏). Unit-type is defined as built-in 𝟏-type. $$ 𝟏 : U. $$

def Unit : U := 𝟏

Constructor

Definition (Star). 𝟏-type containts single inhabitant ★: $$ ★ : 𝟏. $$

def 1-ctor : 𝟏 := ★

Induction Principle

Definition (Induction Principle $ind_1$). 𝟏-type satisfies induction principle that for any family $P(x)$ indexed by $x : 1$ there is a function $$ \prod_{C\ :\ 𝟏\ \rightarrow\ U} \prod_{x\ :\ C(★)} \prod_{z\ :\ 𝟏} C(z) := ind_1(C,x,z). $$

def 1-ind (C: 𝟏 → U) (x: C ★) (z: 𝟏) : C z := ind₁ C x z

def 2-ind (C: 𝟐 → U) (x: C 0₂) (y: C 1₂) (z: 𝟐) : C z := ind₂ C x y z
